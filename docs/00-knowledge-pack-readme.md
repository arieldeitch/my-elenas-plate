# Nutrition Tracker Knowledge Pack

## מטרת המסמכים

ספרייה זו היא מקור הידע המרכזי של פרויקט אפליקציית מעקב התזונה המשותפת.

היא נועדה לאפשר עבודה רציפה ומתואמת בין:

- המשתמש
- ה-GPT
- Lovable
- GitHub
- Claude Code
- Supabase
- כלי AI נוספים

אין להסתמך על זיכרון של שיחה, סשן או כלי AI כמקור המידע היחיד.

כל ידע מהותי הנדרש להמשך העבודה צריך להישמר ב-repository.

---

## תחזוקת המסמכים

יש לעדכן מסמך כאשר חל שינוי מהותי בתחום שהוא מתאר.

אין לשכפל מידע שלא לצורך.

כאשר מידע מופיע בכמה מסמכים:

- מסמך הדרישות מגדיר מה נדרש.
- מסמך הארכיטקטורה מגדיר כיצד המערכת בנויה.
- מסמך ההחלטות מסביר מדוע נבחר פתרון.
- מסמך המצב מתאר היכן הפרויקט נמצא כעת.
- מסמך ה-Handover מסביר כיצד להמשיך מהנקודה הנוכחית.

---

## Definition of Ready

משימה מוכנה לפיתוח כאשר:

- המטרה שלה ברורה.
- הדרישות הרלוונטיות ידועות.
- ניתן לזהות את הקבצים או הרכיבים המושפעים.
- אין החלטה עסקית חסרה.
- קיימים Acceptance Criteria בסיסיים.

אין לעצור פיתוח בגלל פרט טכני שניתן להחליט עליו לפי Best Practices.

---

## Definition of Done

משימה נחשבת מושלמת כאשר:

- הדרישה מומשה.
- הקוד נבנה ללא שגיאות.
- אין Type Errors חדשים.
- אין שגיאות Lint מהותיות.
- לא נוצרה Regression ידועה.
- הנתונים נשמרים ומסתנכרנים כנדרש.
- מצבי Loading, Empty ו-Error טופלו.
- התוצאה תקינה ב-Mobile.
- התיעוד הרלוונטי עודכן.
- `project-status.md` ו-`todo.md` משקפים את המצב החדש.
- בוצע Commit כאשר הדבר מתאים לזרימת העבודה.
