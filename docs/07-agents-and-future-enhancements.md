# Agents and Future Enhancements

## 1. מטרת המסמך

מסמך זה שומר רעיונות שאינם חלק מה-MVP הנוכחי.

המטרה היא:

- למנוע הרחבת Scope במהלך הפיתוח
- לשמר רעיונות טובים להמשך
- להבחין בין תשתית עתידית לבין Feature מאושר
- לאפשר תכנון מסודר לאחר שה-MVP יציב

שום פריט במסמך זה אינו מאושר לפיתוח אוטומטית.

---

## 2. תנאי כניסה ל-Roadmap

Feature עתידי יישקל רק לאחר:

- שימוש אמיתי ב-MVP
- אימות שה-Flow הבסיסי נוח
- יציבות שמירה וסנכרון
- איכות נתונים מספקת
- הבנת צורך אמיתי של שני המשתמשים
- הערכת פרטיות ואבטחה

---

## 3. Dashboard ו-Analytics

אפשרויות עתידיות:

- מגמת משקל
- מגמת אחוז שומן
- מסת שומן לאורך זמן
- תדירות תיעוד
- שיעור ימים מלאים וחלקיים
- מזונות נפוצים
- התפלגות ארוחות
- השוואה שבועית וחודשית
- קשר בין אימון לתיעוד
- קשר בין צום לתיעוד
- השוואה בין תקופות

העקרונות:

- אין להסיק מסקנות רפואיות ללא בסיס מתאים
- אין להציג Correlation כ-Causation
- אין לייצר ציונים שיפוטיים
- יש להציג נתונים באופן פשוט ונגיש

## 4. יעדים

אפשרויות עתידיות:

- יעד משקל
- יעד תדירות תיעוד
- יעד חלון צום
- יעד אישי לשגרת ארוחות

הוספת יעד מחייבת שינוי במודל הטון והצבעים.

לדוגמה, ירידה במשקל אינה תמיד יעד ולכן צבעי הדלתא צריכים להפוך ל-Goal Aware אם Feature זה נוסף.

## 8. הזנה קולית

Feature עתידי אפשרי:

- משפט טבעי כגון: `אכלתי שתי פרוסות לחם, ביצה וסלט`
- פירוק לפריטי מזון
- הצעת כמויות
- אישור לפני שמירה

יש לתמוך בעברית ולהימנע משמירה שגויה ללא Review.

## 9. Agents עתידיים

### 9.1 Data Quality Agent

תפקיד אפשרי:

- זיהוי כפילויות במאגר מזון
- הצעת איחוד שמות
- זיהוי Units חריגים
- איתור רשומות חסרות או לא עקביות

ה-Agent לא יבצע מחיקה או איחוד ללא אישור.

### 9.2 Weekly Summary Agent

תפקיד אפשרי:

- סיכום עובדתי של השבוע
- הצגת ימים חסרים
- הצגת מגמות משקל, אם יש מספיק נתונים
- הצגת מזונות או ארוחות נפוצים

ה-Agent לא ייתן ייעוץ רפואי או ציון.

### 9.3 Logging Assistant

תפקיד אפשרי:

- קבלת טקסט או קול
- הצעת רשומות מזון
- השלמת יחידות נפוצות
- שימוש ב-Recent Foods

כל פעולה תדרוש Review ברור לפני Commit לנתונים.

### 9.4 Insight Agent

תפקיד אפשרי:

- זיהוי דפוסים לאורך זמן
- הצעת שאלות לבדיקה
- הצגת Correlations בזהירות

אין להציג אבחנה, המלצה רפואית או סיבתיות.

## 16. Feature Review Template

לפני אישור Feature עתידי יש לענות:

- מה הבעיה שהוא פותר?
- מי צריך אותו?
- מה הראיות לצורך?
- האם הוא מוסיף חיכוך?
- האם הוא דורש מידע רגיש נוסף?
- האם הוא משנה את ה-Scope?
- מה מדד ההצלחה?
- מהו ה-MVP של ה-Feature?
- אילו נתונים או Migrations נדרשים?
- האם ניתן להסירו בקלות אם אינו מועיל?
